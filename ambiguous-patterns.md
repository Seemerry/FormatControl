# 歧义语句检测条件列表

当语法修复无法确定作者意图时，以下模式会被标记为"歧义"并路由到 LLM 进行语义判断。

检测逻辑位于 `src/lib/remend-blocks.ts` 的 `findAmbiguousPatterns()` 函数中。

---

## A. 不对称强调标记

### A1. 星号 1 vs 2 不对称

- **正则**：`/(?<!\*)\*{2}(?!\*)((?:(?!\*).)+?)\*(?!\*)/`
- **触发条件**：一侧 `**`，另一侧 `*`，且前后无更多 `*`
- **示例**：`**文本*`、`*文本**`
- **歧义**：加粗缺闭合（→ `**文本**`）还是斜体多了一个 `*`（→ `*文本*`）

### A2. 星号 3 vs 1/2 不对称

- **正则**：`/(?<!\*)\*{3}(?!\*)((?:(?!\*{1,3}).)+?)\*{1,2}(?!\*)/`
- **触发条件**：开头 `***`，结尾 `*` 或 `**`
- **示例**：`***文本**`、`***文本*`
- **歧义**：粗斜体缺闭合（→ `***文本***`）、加粗多了 `*`（→ `**文本**`）、还是斜体多了 `**`（→ `*文本*`）

### A3. 下划线 1 vs 2 不对称

- **正则**：`/(?<!_)_{1,2}(?!_)((?:(?!_).)+?)_{1,2}(?!_)/`
- **触发条件**：两侧 `_` 数量不等（如 `_文本__` 或 `__文本_`）
- **歧义**：同 A1，加粗 vs 斜体方向不确定

## B. $ 符号（数学公式 vs 货币）

- **正则**：`/(?<!\$)\$(?!\$)([^$\n]+)/`
- **触发条件**：
  1. 行内 `$` 后无配对的闭合 `$`（未闭合）
  2. 且内容满足以下任一：以数字开头（疑似货币），或含 `= ^ \ { } ∑ ∫` 等数学字符
- **示例**：`$50`（货币）、`$E=mc^2`（公式）
- **歧义**：货币符号保持不变 vs 数学公式需要闭合 `$`

---

## C. 图片链接缺少 ! 前缀（image vs link）

- **正则**：`/(?<!!)\[([^\]]+)\]\((https?:\/\/[^)]+\.(?:png|jpe?g|gif|svg|webp|bmp|ico))\)/i`
- **触发条件**：
  1. `[描述](URL)` 格式（非 `![` 开头）
  2. URL 以图片扩展名结尾（png / jpg / gif / svg / webp / bmp / ico）
- **示例**：`[架构图](https://example.com/arch.png)`
- **歧义**：忘写 `!` 的图片（→ `![架构图](...)`）vs 有意的指向图片文件的超链接

---

## D. 算术/标识符中的格式字符

### D1. 数字间的星号

- **正则**：`/(\d)\*([^*\s]+)\*(\d)/`
- **触发条件**：`数字*内容*数字` 模式
- **示例**：`3*4*5`
- **歧义**：`*4*` 被渲染为斜体 vs 实际是乘法运算（需转义为 `3\*4\*5`）

### D2. 标识符中的下划线

- **正则**：`/\w_([\w]+(?:\/[\w]+)*)_\w/`
- **触发条件**：`字符_内容_字符` 模式，且看起来像文件路径或 snake_case
- **示例**：`my_project_name`、`/home/user/my_file`
- **歧义**：`_project_` 被渲染为斜体 vs 实际是标识符的一部分（需转义为 `my\_project\_name`）

---

## E. 未闭合代码块边界

- **触发条件**（多行状态检测）：
  1. 存在 `` ``` `` 或 `~~~` 开头的代码块
  2. 代码块未正常闭合
  3. 代码块内出现空行
  4. 空行之后出现 Markdown 语法特征
- **语法特征匹配正则**：`/^(#{1,6}\s|[-*+]\s|\d+\.\s|>\s|!\[|\[.+\]\()/`
  - 标题（`## `）、无序列表（`- `、`* `、`+ `）、有序列表（`1. `）、引用（`> `）、图片（`![`）、链接（`[text](`）
- **示例**：
  ```
  ```python
  print("hello")

  ## 接下来的正文
  ```
- **歧义**：代码块应在空行处结束（后面是正文）vs 空行和后续内容都属于代码块

---

## F. 反引号定界符分组

- **正则**：`` /`[^`]*`[^`]*`/ ``
- **触发条件**：匹配中含 3 个以上反引号（多个 `` ` `` 交错出现）
- **示例**：`` `let x = `hello`` ``
- **歧义**：哪些反引号是定界符、哪些是内容无法确定

---

## 全局规则

| 规则 | 说明 |
|------|------|
| 代码块内跳过 | 已闭合代码块内的所有内容不做歧义检测 |
| 附带上下文 | 每个歧义段附带 `original`（原文）、`context`（前后文）、`syntacticFix`（语法修复结果） |
| 最小发送 | 仅将歧义片段发送给模型，非歧义部分直接使用语法修复结果 |
| 可开关 | 通过配置面板"修复模型"tab 的开关控制是否启用语义修复 |
